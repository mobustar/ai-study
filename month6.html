VAE (変分オートエンコーダ) Day 103–107

「新しいデータを生成する」モデルを作るにはどうすればよいでしょうか？単純なオートエンコーダ（AE）は入力を圧縮して再構成できますが、潜在空間が不連続なため「ランダムサンプリングして生成」することが難しいです。VAEは潜在変数を「確率分布」として表現することでこの問題を解決し、滑らかな潜在空間を実現します。

VAE のイメージ：「言語に翻訳してから元に戻す機械」

VAEは「データ → 意味（潜在変数）→ データ」という圧縮・展開を学習します。手書き数字「8」を入力すると、エンコーダは「縦長？横長？太い？細い？傾いてる？」のような意味的な座標（約20次元）に圧縮します。デコーダはその座標から「8」を再生成します。

潜在空間の座標Aと座標Bの中間点を取ると「AとBの中間の見た目」が生成されます（補間）。また座標を完全にランダムにサンプリングすれば新しい「それらしい8」が生成できます。

ELBOを導出する

データの対数尤度 $\log p(x)$ を直接最大化することは難しいため、その下界（ELBO）を最大化します。まず $\log p(x)$ を分解します：

$$\log p(x) \geq \underbrace{\mathbb{E}_{q(z|x)}[\log p(x|z)]}_{\text{再構成精度（再構成損失）}} - \underbrace{D_{KL}(q(z|x) \| p(z))}_{\text{正則化（潜在空間を標準正規に近づける）}}$$

エンコーダ $q(z|x)$ は入力から潜在変数の分布 $\mathcal{N}(\mu, \sigma^2)$ を出力します。デコーダ $p(x|z)$ はサンプリングした $z$ から元のデータを復元します。

再パラメータ化トリック — 「ランダム性を外に出す」

$$z = \mu + \sigma \odot \epsilon, \quad \epsilon \sim \mathcal{N}(0, I)$$
なぜ再パラメータ化が必要か

$z \sim \mathcal{N}(\mu, \sigma^2)$ からの「サンプリング」操作は微分不可能です。しかし $z = \mu + \sigma \odot \epsilon$ と書き換えると、$z$ は $\mu$ と $\sigma$ の決定論的な関数になり、通常の逆伝播が可能になります。ランダム性は外部変数 $\epsilon$ に「押し出す」のがこのトリックの本質です。

Python — VAE の完全実装（定義・学習・生成・補間） 
import torch
import torch.nn as nn
import torch.nn.functional as F
import torch.optim as optim

class VAE(nn.Module):
    """Variational Autoencoder (MNIST用: 784次元 → 潜在20次元 → 784次元)"""
    def __init__(self, input_dim=784, latent_dim=20):
        super().__init__()
        # ─── エンコーダ: x → (μ, log_var) ───
        # 入力画像を圧縮して、潜在分布のパラメータを出力する
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, 400),
            nn.ReLU()
        )
        # μ: 潜在分布の平均
        self.fc_mu      = nn.Linear(400, latent_dim)
        # log σ²: 潜在分布の分散の対数（分散=exp(log_var)は常に正になる）
        self.fc_log_var = nn.Linear(400, latent_dim)

        # ─── デコーダ: z → x_recon ───
        # 潜在変数から元の画像を復元する
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, 400),
            nn.ReLU(),
            nn.Linear(400, input_dim),
            nn.Sigmoid()   # 出力を[0,1]に変換（ピクセル値に対応）
        )

    def encode(self, x):
        """入力を潜在分布のパラメータに変換"""
        h = self.encoder(x)                  # (B, 784) → (B, 400)
        mu      = self.fc_mu(h)              # (B, 400) → (B, 20) 平均
        log_var = self.fc_log_var(h)         # (B, 400) → (B, 20) 対数分散
        return mu, log_var

    def reparameterize(self, mu, log_var):
        """
        再パラメータ化トリック: z = μ + σ × ε, ε ~ N(0,I)
        ポイント: サンプリングを「外の ε」に押し出すことで
                  z から μ, σ への勾配が流れるようになる
        """
        if self.training:
            std = torch.exp(0.5 * log_var)   # σ = exp(log_var / 2)
            eps = torch.randn_like(std)       # ε ~ N(0, I)
            return mu + std * eps             # z ~ N(μ, σ²)
        else:
            return mu  # 推論時はノイズなし（μをそのまま使う）

    def decode(self, z):
        """潜在変数から画像を復元"""
        return self.decoder(z)               # (B, 20) → (B, 784)

    def forward(self, x):
        mu, log_var = self.encode(x)
        z = self.reparameterize(mu, log_var)
        x_recon = self.decode(z)
        return x_recon, mu, log_var

def vae_loss(x, x_recon, mu, log_var, beta=1.0):
    """
    ELBO損失 = 再構成損失 + β × KL損失
    beta=1: 標準VAE, beta>1: disentangled表現を促進(β-VAE)
    """
    # 再構成損失: 元画像と復元画像のBCE（pixel単位）
    recon_loss = F.binary_cross_entropy(x_recon, x, reduction='sum')

    # KL損失: q(z|x) = N(μ,σ²) と p(z) = N(0,I) の KLダイバージェンス
    # 解析解: -0.5 × Σ(1 + log σ² - μ² - σ²)
    kl_loss = -0.5 * torch.sum(1 + log_var - mu.pow(2) - log_var.exp())

    return recon_loss + beta * kl_loss

# ─── 学習ループ（1エポック分のサンプル）───
vae = VAE(input_dim=784, latent_dim=20)
optimizer = optim.Adam(vae.parameters(), lr=1e-3)

for step in range(3):   # 実際はデータローダーで全データを繰り返す
    x = torch.rand(32, 784)    # 仮：32枚のMNIST画像（0〜1に正規化済み）

    x_recon, mu, log_var = vae(x)
    loss = vae_loss(x, x_recon, mu, log_var)

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    print(f"step {step}: loss={loss.item():.1f}")

# ─── 新しい画像を生成（推論）───
vae.eval()
with torch.no_grad():
    # N(0,I) からサンプリング → デコーダで生成
    z_sample = torch.randn(4, 20)          # 4枚分のランダムな潜在変数
    generated = vae.decode(z_sample)        # → (4, 784) ← 生成された画像！
    print(f"生成画像 shape: {generated.shape}")

    # 2枚の画像の中間を補間（潜在空間の滑らかさを確認）
    img_a = torch.rand(1, 784)
    img_b = torch.rand(1, 784)
    mu_a, _ = vae.encode(img_a)
    mu_b, _ = vae.encode(img_b)
    # α を 0 → 1 に変化させて中間の画像を生成
    for alpha in [0.0, 0.25, 0.5, 0.75, 1.0]:
        z_interp = (1 - alpha) * mu_a + alpha * mu_b  # 線形補間
        img = vae.decode(z_interp)
        print(f"α={alpha:.2f}: 補間画像 mean={img.mean():.3f}")

KL項の解析解（正規分布の場合）

$$D_{KL}(\mathcal{N}(\mu, \sigma^2) \| \mathcal{N}(0, 1)) = -\frac{1}{2}\sum_{j=1}^{J}(1 + \log\sigma_j^2 - \mu_j^2 - \sigma_j^2)$$
この式は解析的に計算できる（数値積分不要）ため、実装が非常に簡潔です。$\mu \to 0, \sigma \to 1$ のとき KL = 0（標準正規分布に一致）になります。
学習時

損失 = 再構成損失（例: MSEまたはBCE）+ $\beta \cdot D_{KL}$ 項。$\beta > 1$ に設定すると潜在空間の分離性が向上します（$\beta$-VAE）。エンコーダ・デコーダを同時にエンドツーエンドで学習します。

推論時（生成）

$z \sim \mathcal{N}(0, I)$ からサンプリングしてデコーダに通すだけで新しいデータを生成できます。潜在空間での内挿（$z_1$ と $z_2$ の中点を取る）によりスムーズな変換が可能です。

本節のまとめ
  • VAEは潜在変数を「点」でなく「分布（μ, σ²）」として表現することで滑らかな潜在空間を作る
  • ELBO = 再構成損失 + KL損失（正則化）の最大化が目標
  • 再パラメータ化トリック：z = μ + σ×ε とすることでサンプリングを微分可能にする
  • 学習後は z～N(0,I) からサンプリングするだけで新しいデータを生成できる

GAN (敵対的生成ネットワーク) Day 108–112

VAEは損失関数の設計が明確でしたが、再構成画像がぼやける傾向がありました。GANは全く異なるアプローチをとります。「本物と偽物を見分ける識別器」と「識別器を騙すような偽物を生成する生成器」を競わせることで、驚くほどリアルな画像を生成できます。ただし学習が不安定という大きな課題もあります。

GAN のイメージ：「偽造師と探偵のゲーム」

Generator（生成器）は「完璧な偽札を作る偽造師」です。Discriminator（識別器）は「本物と偽物を見抜く探偵」です。

・探偵は「本物の1000円札」と「偽造師の作品」を区別するよう訓練される
・偽造師は「探偵を騙せる品質の偽札」を作れるよう訓練される
・これを繰り返すと偽造師の技術が限りなく向上し、最終的には探偵でさえ区別できなくなる（Nash均衡）

ミニマックスゲームの目的関数

$$\min_G \max_D \; \mathbb{E}_{\mathbf{x} \sim p_{data}}[\log D(\mathbf{x})] + \mathbb{E}_{\mathbf{z} \sim p_z}[\log(1 - D(G(\mathbf{z})))]$$

Discriminatorは本物を1、偽物を0に分類しようとし（max）、Generatorは偽物を1と判定させようとします（min）。

最適な Discriminator は $D^*(\mathbf{x}) = \frac{p_{data}(\mathbf{x})}{p_{data}(\mathbf{x}) + p_g(\mathbf{x})}$ となります。Nash均衡では $p_g = p_{data}$ のとき $D^*(\mathbf{x}) = \frac{1}{2}$（本物と偽物を区別できなくなる）です。
Python — GAN の定義から学習ループまで（MNIST用） 
import torch
import torch.nn as nn
import torch.optim as optim

# ─── Generator: ノイズ z → 偽画像（784次元）───
class Generator(nn.Module):
    """ランダムなノイズから手書き数字のような画像を生成する"""
    def __init__(self, latent_dim=100, output_dim=784):
        super().__init__()
        self.net = nn.Sequential(
            # 100次元のノイズを徐々に拡張していく
            nn.Linear(latent_dim, 256),
            nn.LeakyReLU(0.2),          # GANでよく使われる活性化関数
            nn.BatchNorm1d(256),
            nn.Linear(256, 512),
            nn.LeakyReLU(0.2),
            nn.BatchNorm1d(512),
            nn.Linear(512, output_dim),
            nn.Tanh()                   # 出力を[-1, 1]に正規化（画像に合わせる）
        )

    def forward(self, z):
        return self.net(z)              # (batch, latent) → (batch, 784)

# ─── Discriminator: 画像 x → 本物確率（0〜1）───
class Discriminator(nn.Module):
    """画像が本物か偽物かを見分ける識別器"""
    def __init__(self, input_dim=784):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, 512),
            nn.LeakyReLU(0.2),          # 勾配が完全にゼロにならないLeakyReLU
            nn.Dropout(0.3),             # 過学習防止
            nn.Linear(512, 256),
            nn.LeakyReLU(0.2),
            nn.Dropout(0.3),
            nn.Linear(256, 1),
            nn.Sigmoid()                 # 本物確率（0=偽物, 1=本物）
        )

    def forward(self, x):
        return self.net(x)              # (batch, 784) → (batch, 1)

# ─── モデルとオプティマイザの準備 ───
G = Generator(latent_dim=100, output_dim=784)
D = Discriminator(input_dim=784)

# GAN でよく使われる設定: lr=2e-4, β1=0.5（Adam のデフォルトは0.9）
opt_G = optim.Adam(G.parameters(), lr=2e-4, betas=(0.5, 0.999))
opt_D = optim.Adam(D.parameters(), lr=2e-4, betas=(0.5, 0.999))

criterion = nn.BCELoss()   # 二値交差エントロピー（本物/偽物の分類）

# ─── 学習ループ（1エポック分のサンプル）───
# 実際は dataloader からデータを受け取るが、ここでは仮データで説明
for step in range(3):   # 実際は全データを繰り返す
    B = 32   # バッチサイズ
    real_batch = torch.randn(B, 784)  # 仮：本物データ（MNISTなら実際の画像）

    ones  = torch.ones(B, 1)     # 本物ラベル = 1
    zeros = torch.zeros(B, 1)    # 偽物ラベル = 0

    # ━━━ Step 1: Discriminator を更新（D を賢くする）━━━
    z    = torch.randn(B, 100)   # ランダムノイズ
    fake = G(z).detach()         # ← detach()が重要！Gへの勾配を切断

    # 本物画像に「1（本物）」、偽物画像に「0（偽物）」を予測させる
    loss_real = criterion(D(real_batch), ones)
    loss_fake = criterion(D(fake),       zeros)
    loss_D    = (loss_real + loss_fake) / 2    # 両方の損失の平均

    opt_D.zero_grad()
    loss_D.backward()
    opt_D.step()

    # ━━━ Step 2: Generator を更新（G を賢くする）━━━
    z = torch.randn(B, 100)
    # Gの目標: Dが偽物を「本物（1）」と判定するよう騙す
    loss_G = criterion(D(G(z)), ones)  # 偽物に「本物ラベル=1」で学習

    opt_G.zero_grad()
    loss_G.backward()            # ← detach() しないので G まで勾配が伝わる
    opt_G.step()

    print(f"step {step}: loss_D={loss_D.item():.4f}, loss_G={loss_G.item():.4f}")

# 新しい画像を生成（推論）
G.eval()
with torch.no_grad():
    z_sample = torch.randn(4, 100)   # 4枚生成
    generated = G(z_sample)          # (4, 784) ← 偽画像！
    print(f"生成画像 shape: {generated.shape}")
GANの問題とWGAN

実際には学習が不安定で、Mode Collapse（特定のデータのみ生成するようになる多様性の喪失）が大きな問題です。
WGAN：Jensen-Shannonダイバージェンスの代わりにWasserstein（Earth Mover）距離を使うことで勾配消失とMode Collapseを大幅に改善。Discriminatorを「Critic」として重みをクリップ（またはGradient Penalty）します。

学習時の注意点

GeneratorとDiscriminatorを交互に更新します。通常Discriminatorを先に数ステップ更新してからGeneratorを1ステップ更新します。学習率は非常に小さく（例: $2 \times 10^{-4}$）、バランスが崩れると片方が支配して学習崩壊します。

本節のまとめ
  • GAN = Generator（偽造師）とDiscriminator（探偵）の競争によって学習する
  • Nash均衡では生成分布が真のデータ分布と一致し、識別確率が0.5になる
  • GとDを交互に更新。学習率・更新比率のバランスが崩れると学習崩壊する
  • Mode Collapse問題を解決するためWGANなどの改良手法が重要

Diffusion Model Day 112

GANは高品質な画像を生成できますが、学習が不安定です。Diffusion Modelは全く異なる発想で高品質生成を実現します。「画像にノイズを少しずつ加えて完全なノイズにする」プロセスを学習し、逆に「ノイズから画像を復元する」ことを学習します。DALL-E 2、Stable Diffusion、Midjourneyの基盤技術です。

Diffusion Model のイメージ：「砂嵐からの逆フィルム」

きれいな写真（$\mathbf{x}_0$）に少しずつ「砂嵐（ノイズ）」を加え続けると、最終的には純粋なランダムノイズ（$\mathbf{x}_T$）になります（前方拡散：$T$ステップ）。

Diffusion Modelは「この砂嵐が1ステップ前にどんな画像だったか」を予測することを学習します。これを繰り返し（逆拡散：$T$ → $0$）、最初の純粋なランダムノイズからきれいな画像を復元します。
驚くべきことに「砂嵐の各ステップで何のノイズが加わったか」を予測するだけで、完璧な画像生成が可能になります。

前方拡散過程（Forward Process）

元画像 $\mathbf{x}_0$ に少しずつノイズを加えて純粋なガウスノイズに変換します：

$$q(\mathbf{x}_t | \mathbf{x}_{t-1}) = \mathcal{N}(\mathbf{x}_t; \sqrt{1-\beta_t}\mathbf{x}_{t-1}, \beta_t I)$$

$\bar{\alpha}_t = \prod_{s=1}^t (1-\beta_s)$ とすると、任意のステップ $t$ のノイズを一撃で計算できます（閉形式）：

$$q(\mathbf{x}_t | \mathbf{x}_0) = \mathcal{N}(\mathbf{x}_t; \sqrt{\bar{\alpha}_t}\mathbf{x}_0, (1-\bar{\alpha}_t) I)$$

学習目的 — 「どのノイズが加わったか」を予測する

$$\mathcal{L} = \mathbb{E}_{t, \mathbf{x}_0, \epsilon}\left[\|\epsilon - \epsilon_\theta(\mathbf{x}_t, t)\|^2\right]$$
Python — Diffusion Model の前方拡散・学習・逆拡散を段階的に実装 
import torch
import torch.nn as nn
import numpy as np

# ─── ノイズスケジュールの設定 ───
T = 1000   # 合計ステップ数（DDPM論文の設定）

# β_t: 各ステップで加えるノイズの量（線形スケジュール）
betas = torch.linspace(1e-4, 0.02, T)   # 0.0001 〜 0.02 に線形増加

# ᾱ_t = Π_{s=1}^{t} (1 - β_s)：t ステップ後の「元画像の残量」
alphas      = 1 - betas
alphas_bar  = torch.cumprod(alphas, dim=0)   # 累積積

# 閉形式に使う係数を事前計算
sqrt_ab           = torch.sqrt(alphas_bar)            # √ᾱ_t
sqrt_one_minus_ab = torch.sqrt(1 - alphas_bar)        # √(1-ᾱ_t)

print(f"t=0  のとき √ᾱ = {sqrt_ab[0]:.4f}（元画像 ≈ そのまま）")
# → 0.9999（ほぼノイズなし）
print(f"t=999のとき √ᾱ = {sqrt_ab[-1]:.4f}（元画像 ≈ 消えている）")
# → 0.0077（ほぼ純粋なノイズ）

# ─── 前方拡散：閉形式で任意ステップの x_t を一発生成 ───
def forward_diffusion(x0, t):
    """
    x0: (B, C, H, W) — 元の画像
    t:  (B,)         — 各サンプルのタイムステップ
    """
    eps = torch.randn_like(x0)   # ε ~ N(0, I)

    # x_t = √ᾱ_t × x_0 + √(1-ᾱ_t) × ε
    # ↑ 元画像の成分   ↑ ノイズの成分
    ab  = sqrt_ab[t].view(-1, 1, 1, 1)           # shape を合わせる
    oab = sqrt_one_minus_ab[t].view(-1, 1, 1, 1)
    x_t = ab * x0 + oab * eps

    return x_t, eps    # x_t と「加えたノイズ ε」を返す

# ─── 学習ループ（1ステップ）───
def train_step(model, x0, optimizer):
    """
    model: ノイズを予測するネットワーク（U-Net など）
           入力: (x_t, t) → 出力: ε_pred（予測ノイズ）
    """
    B = x0.shape[0]

    # ① ランダムなタイムステップを選択（毎回異なる t で学習）
    t = torch.randint(0, T, (B,))

    # ② 閉形式でノイズを加えた x_t を生成
    x_t, eps_true = forward_diffusion(x0, t)

    # ③ モデルに「このノイズ x_t を作るために加えた ε は？」と聞く
    eps_pred = model(x_t, t)

    # ④ 予測ノイズと実際のノイズの差（MSE）を最小化
    loss = ((eps_true - eps_pred) ** 2).mean()
    # ↑ 驚くほどシンプル！GAN の複雑な学習ループと比べると格段に安定

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    return loss.item()
学習時のアルゴリズム

① ランダムなタイムステップ $t \sim \text{Uniform}(1, T)$ をサンプリング（$T$ は通常1000）
② $\mathbf{x}_t = \sqrt{\bar{\alpha}_t}\mathbf{x}_0 + \sqrt{1-\bar{\alpha}_t}\epsilon$ で一撃でノイズ追加（閉形式を活用）
③ $\epsilon_\theta(\mathbf{x}_t, t)$ でノイズを予測し MSE 損失で更新
単純な MSE 損失で安定学習できることが GAN に対する大きな利点です。

推論時 — 逆拡散の逐次デノイズ

$\mathbf{x}_T \sim \mathcal{N}(0, I)$ から出発し、$T$ ステップかけてノイズを除去します：

$$\mathbf{x}_{t-1} = \frac{1}{\sqrt{\alpha_t}}\left(\mathbf{x}_t - \frac{1-\alpha_t}{\sqrt{1-\bar{\alpha}_t}}\epsilon_\theta(\mathbf{x}_t, t)\right) + \sigma_t \mathbf{z}$$

DDPM: $T=1000$ ステップ → 高品質だが生成が遅い
DDIM: 決定論的サンプリングで $T=20\sim50$ ステップに削減可能
Classifier-Free Guidance (CFG): 条件あり/なし予測を補間してクオリティと多様性を制御：$\hat{\epsilon} = \epsilon_\theta(\mathbf{x}_t) + w \cdot (\epsilon_\theta(\mathbf{x}_t, c) - \epsilon_\theta(\mathbf{x}_t))$（$w$ はガイダンス強度）

VAE/GANとの比較
学習安定性画像品質多様性生成速度
VAE安定ぼやけがち高速（1ステップ）
GAN不安定高品質低（Mode Collapse）速（1ステップ）
Diffusion非常に安定最高品質高遅（$T$ステップ）
本節のまとめ
  • 前方拡散：画像にノイズを少しずつ加えて純粋ノイズに変換。閉形式で任意ステップを一発計算
  • 学習：「加えたノイズ ε を予測する」というシンプルなMSE損失で安定学習
  • 逆拡散：純粋ノイズからTステップかけてノイズを取り除いて画像を生成
  • DDIM で推論ステップを大幅削減。CFGで品質と多様性をトレードオフ制御

強化学習 Day 113–117

教師あり学習は「正解データ」が必要ですが、強化学習は「報酬」だけを手掛かりに学習します。チェスや囲碁では全ての局面に「正解手」を定義できませんが、「勝ち負け」という報酬は明確です。AlphaGoは強化学習でプロ棋士を圧倒し、ChatGPTもRLHF（人間のフィードバックからの強化学習）で「人に役立つ回答」を学習しています。

強化学習のイメージ：「ゲームで高得点を目指すAI」

強化学習のエージェントはゲームプレイヤーのように行動します。
・状態（State）$s$：現在のゲーム画面
・行動（Action）$a$：上下左右への移動
・報酬（Reward）$r$：コインを取ると+1、死ぬと-100
・方策（Policy）$\pi$：状態を見て行動を決めるルール

エージェントは「今の報酬」だけでなく「将来の累積報酬」を最大化しようとします。これがベルマン方程式の核心です。

ベルマン方程式 — 「今の価値 = 今の報酬 + 未来の価値」

$$V^*(s) = \max_a \left[R(s, a) + \gamma \sum_{s'} P(s'|s,a) V^*(s')\right]$$
$$Q^*(s, a) = R(s, a) + \gamma \sum_{s'} P(s'|s, a) \max_{a'} Q^*(s', a')$$

$\gamma \in [0,1)$ は割引率（将来の報酬をどれだけ重視するか）。これらは再帰的な方程式で、動的計画法や近似（DQN等）で解きます。

Q学習 — ベルマン方程式を反復で解く

$$Q(s, a) \leftarrow Q(s, a) + \alpha \left[\underbrace{r + \gamma \max_{a'} Q(s', a')}_{\text{TD目標}} - Q(s, a)\right]$$
Python — Q学習をゼロから実装（迷路を解くエージェント） 
import numpy as np

# ─── 環境の定義（4×4グリッド迷路）───
# 状態 0〜15 の格子世界。15 がゴール
# 行動: 0=上, 1=右, 2=下, 3=左
n_states  = 16    # 4×4 = 16 状態
n_actions = 4     # 上右下左

def step(state, action):
    """行動を実行して次の状態と報酬を返す"""
    row, col = state // 4, state % 4

    if   action == 0: row -= 1   # 上
    elif action == 1: col += 1   # 右
    elif action == 2: row += 1   # 下
    elif action == 3: col -= 1   # 左

    # 壁に当たったらその場に留まる
    row = max(0, min(3, row))
    col = max(0, min(3, col))
    next_state = row * 4 + col

    # 報酬の設定
    if next_state == 15:   reward, done = +1.0, True   # ゴール
    else:                  reward, done = -0.01, False  # 移動コスト

    return next_state, reward, done

# ─── Q テーブルの初期化 ───
Q = np.zeros((n_states, n_actions))  # 全ての状態・行動の価値推定

# ─── ハイパーパラメータ ───
alpha   = 0.1    # 学習率（Q値の更新幅）
gamma   = 0.99   # 割引率（将来の報酬をどれだけ重視するか）
epsilon = 0.1    # 探索率（10%の確率でランダム行動 = 探索）

# ─── ε-greedy 行動選択 ───
def choose_action(state):
    if np.random.random() < epsilon:
        return np.random.randint(n_actions)  # ランダム探索（新しい経路を試す）
    return np.argmax(Q[state])              # 現在の最良行動を選択

# ─── 学習ループ（1000エピソード）───
for episode in range(1000):
    s = 0     # スタート位置（状態0）
    total_reward = 0

    while True:
        a       = choose_action(s)
        s_next, r, done = step(s, a)

        # ─ TD更新（ベルマン方程式の近似）─
        # TD目標: r + γ × max_a Q(s', a')（「もし最適に行動したときの将来価値」）
        td_target = r + gamma * np.max(Q[s_next]) * (not done)
        # TD誤差: 目標値と現在の推定値のズレ
        td_error  = td_target - Q[s, a]
        # Q値を少しずつ目標に近づける
        Q[s, a] += alpha * td_error

        s = s_next
        total_reward += r
        if done:
            break

    if episode % 200 == 0:
        print(f"episode {episode:4d}: total_reward = {total_reward:.3f}")

# ─── 学習結果の確認 ───
print("\n学習済みQ値の最良行動（0=上,1=右,2=下,3=左）:")
best_actions = np.argmax(Q, axis=1)
print(best_actions.reshape(4, 4))   # ゴール(右下)に向かう経路が現れるはず
RLHF (Reinforcement Learning from Human Feedback) の数学

ChatGPT・Claude等の学習プロセスは3段階です：

① SFT（教師あり微調整）：高品質なデモデータで事前学習済み LLM をファインチューニング（通常の交差エントロピー損失）。

② 報酬モデル学習：同じプロンプトへの2つの応答を人間が比較し、ブラッドリー-テリーモデルで確率化：

$$P(y_w \succ y_l) = \sigma(r_\phi(x, y_w) - r_\phi(x, y_l))$$

③ PPO 最適化：報酬最大化 + KL ペナルティで SFT モデルからの逸脱を防ぐ：

$$\max_\pi \mathbb{E}[r_\phi(x,y)] - \beta D_{KL}(\pi \| \pi_{SFT})$$

最近は PPO の複雑さを回避する DPO（Direct Preference Optimization） が主流になりつつあります。選好データから直接ポリシーを最適化します：

$$\mathcal{L}_{DPO} = -\mathbb{E}\left[\log \sigma\left(\beta \log\frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log\frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}\right)\right]$$
本節のまとめ
  • 強化学習は「報酬」だけを手掛かりに「累積報酬を最大化する方策」を学習する
  • ベルマン方程式：「状態の価値 = 今の報酬 + γ × 次の状態の最良価値」という再帰式
  • Q学習はTD誤差を使ってベルマン方程式を反復的に近似する
  • RLHF = SFT → 報酬モデル → PPO の3段階。LLMが人間の好みを学ぶ仕組み
  • DPOはRLHFよりシンプルで、報酬モデル不要。現在主流になりつつある

自然言語処理 Day 118–122

コンピュータはそのままでは文字を「意味」として理解できません。NLPの根本的な問題は「単語を数値ベクトルに変換する」こと、つまりEmbedding（埋め込み）です。「王様 − 男性 + 女性 = 女王様」という有名なベクトル演算が示すように、適切な埋め込みはテキストの意味関係を数値空間に表現できます。

Word2Vec のイメージ：「単語の意味を座標で表す地図」

「東京」「大阪」「京都」が地図上の位置で表せるように、単語の「意味」も高次元空間の座標で表せます。
「dog（犬）」と「cat（猫）」は意味が近いので座標も近い。「dog」と「democracy（民主主義）」は意味が遠いので座標も遠い。

さらにすごいのが：king（王）- man（男）+ woman（女）≈ queen（女王）という「ベクトル演算で意味の計算ができる」ことです。これは分布仮説（「文脈が似た単語は似た意味を持つ」）の力です。

Word2Vec (Skip-gram) — 「周辺単語の予測から意味を学ぶ」

「周辺の単語を予測する」タスクを解くことで単語の分散表現を学習します：

$$P(w_o | w_c) = \frac{\exp(\mathbf{v}_{w_o}' \cdot \mathbf{v}_{w_c})}{\sum_{w \in V} \exp(\mathbf{v}_w' \cdot \mathbf{v}_{w_c})}$$

Python — Language Model の Temperature Sampling を実装する 
import torch
import torch.nn.functional as F

def sample_next_token(logits, temperature=1.0, top_k=None, top_p=None):
    """
    次のトークンをサンプリングする（LLMの推論デコード）

    Args:
        logits:      モデルが出力した生スコア (vocab_size,)
        temperature: < 1 で確信が強まり（sharp）、> 1 で多様性が上がる（flat）
        top_k:       上位k個のトークンのみ残して選択
        top_p:       累積確率がp以上になるまでのトークンから選択（nucleus sampling）
    """
    # ① Temperature でスケール
    logits = logits / temperature   # temperature=0.7: より確信が強くなる

    # ② Top-k フィルタリング
    if top_k is not None:
        # 上位 k 個以外を -∞ にする
        top_k_values, _ = torch.topk(logits, top_k)
        threshold = top_k_values[-1]   # k 番目の値が閾値
        logits = torch.where(logits >= threshold, logits,
                             torch.full_like(logits, float('-inf')))

    # ③ Top-p (Nucleus) フィルタリング
    if top_p is not None:
        probs_sorted, sorted_idx = torch.sort(F.softmax(logits, dim=-1),
                                              descending=True)
        cumsum = torch.cumsum(probs_sorted, dim=-1)
        # 累積確率が top_p を超えた位置以降を除去
        remove_mask = cumsum - probs_sorted > top_p
        probs_sorted[remove_mask] = 0.0
        # 元のインデックス順に戻す
        logits.scatter_(0, sorted_idx, probs_sorted)

    # ④ Softmax で確率に変換してサンプリング
    probs = F.softmax(logits, dim=-1)
    return torch.multinomial(probs, num_samples=1).item()

学習時：言語モデルの事前学習

GPT系：次トークン予測（Causal LM）。全トークンを並列に損失計算できるため非常に効率的。BERT系：マスクトークン予測（MLM）+ 次文予測（NSP）。どちらもトークンのインデックスを教師ラベルとした交差エントロピー損失を使います。

推論時：テキスト生成のデコード戦略

Greedy: 常に最高確率のトークンを選択。速いが単調になりがち。
Beam Search: 上位 $k$ 候補を保持して最高確率の系列を探索。翻訳等で有効。
Top-p Sampling: 累積確率が $p$ を超えるまでのトークンからランダムサンプリング。
温度・Top-p・Top-k を組み合わせて生成品質と多様性を制御するのが現代LLMの標準です。

LoRA / QLoRA — パラメータ効率的ファインチューニング

GPT-4クラスのモデル（数百億パラメータ）を全パラメータ更新でファインチューニングするには数百GBのGPUメモリが必要です。LoRA（Low-Rank Adaptation）は「重み行列の変化量を低ランク行列で近似」することで、更新パラメータ数を100分の1以下に削減しながら同等の性能を達成します。QLoRAはさらに4bit量子化と組み合わせ、24GB GPU一枚で70Bモデルのファインチューニングを可能にしました。

LoRAのアイデア：「変化量だけ低ランクで近似する」
  元の重み行列 W (固定・更新しない)
  ┌────────────────────────┐
  │  W ∈ ℝ^{d×d}          │  ← 例: d=4096 → 16M パラメータ
  │  (d=4096なら16Mパラメータ)│
  └────────────────────────┘

  LoRAが学習するのは「差分」ΔW だけ:
  ΔW = B × A      (低ランク近似)
       ↑     ↑
       B∈ℝ^{d×r}  A∈ℝ^{r×d}   r=8 なら
       (4096×8)   (8×4096)    パラメータ数 = 2×4096×8 = 65,536
                               → 元の 0.4% だけ!

  推論時: y = (W + α/r × B×A) x
  α はスケーリング係数 (通常 α=r で相殺)
$$W' = W + \frac{\alpha}{r} BA, \quad A \sim \mathcal{N}(0, \sigma^2),\; B = 0$$

初期化: Aはランダム正規分布、Bはゼロ → 学習開始時は ΔW=0（元の挙動を保持）

Python — LoRAレイヤーのスクラッチ実装 
import torch
import torch.nn as nn
import math

class LoRALinear(nn.Module):
    """
    LoRA付き線形層
    元の重み W は凍結し、低ランク行列 B×A だけ学習する
    """
    def __init__(self, in_features, out_features, r=8, alpha=16):
        super().__init__()
        self.r = r
        self.alpha = alpha
        self.scaling = alpha / r          # スケーリング係数

        # ─── 元の重み（凍結） ───
        self.weight = nn.Parameter(
            torch.randn(out_features, in_features) / math.sqrt(in_features),
            requires_grad=False           # ← 更新しない！
        )
        self.bias = nn.Parameter(torch.zeros(out_features), requires_grad=False)

        # ─── LoRA行列（学習対象） ───
        # A: ランダム初期化（情報を受け取る側）
        self.lora_A = nn.Parameter(torch.randn(r, in_features) * 0.01)
        # B: ゼロ初期化（学習開始時は ΔW=0 を保証）
        self.lora_B = nn.Parameter(torch.zeros(out_features, r))

    def forward(self, x):
        # 元の変換 + LoRAの差分
        base_out = F.linear(x, self.weight, self.bias)
        # ΔW x = B(Ax) を順番に計算（行列積の分解）
        lora_out = (x @ self.lora_A.T) @ self.lora_B.T
        return base_out + self.scaling * lora_out

    def param_count(self):
        total = self.weight.numel() + self.bias.numel()
        lora  = self.lora_A.numel() + self.lora_B.numel()
        print(f"元パラメータ: {total:,}  LoRAパラメータ: {lora:,}  比率: {lora/total:.2%}")

# 動作確認

LoRA が効く理由：「重み変化は低次元多様体に乗っている」

Aghajanyan et al. (2020) の研究で「事前学習済みモデルのファインチューニングに必要な固有次元（intrinsic dimension）は非常に低い」ことが示されました。GPT-3を GLUE タスクにファインチューニングするのに必要な固有次元は僅か数百次元程度です。LoRAはこの性質を利用し、億単位のパラメータの変化を数千〜数万パラメータの低ランク行列で近似しています。

LoRA の重要パラメータ
  • r（ランク）: 低ランク行列の次元。r=4〜64が一般的。大きいほど表現力↑・パラメータ↑。汎用タスクはr=8〜16、専門ドメインはr=32〜64が目安
  • alpha（α）: スケーリング係数。多くの実装で alpha=2×r を推奨。学習率との相互作用に注意
  • target_modules: どの層にLoRAを適用するか。通常はAttentionのQ/V行列。全線形層に適用するとさらに性能↑
  • QLoRAの量子化形式: NF4 > INT8 > INT4（精度順）。NF4はNormalFloat4で正規分布のデータに最適化されており重みデータに最も向いている
まとめ
  • LoRA：重み変化 ΔW を低ランク行列 BA で近似し、学習パラメータを100分の1以下に削減
  • 元の重みは凍結 → 推論時は W+BA をマージして速度劣化なし
  • QLoRA = LoRA + 4bit量子化 → 24GB GPU一枚で70Bモデルのファインチューニングを実現
  • 実用的には PEFT ライブラリを使い、r=16, alpha=32 から始めるのが定石

量子化 — モデルを小さく・速く

通常のニューラルネットワークは32bit浮動小数点（fp32）で保存されますが、量子化によって重みを8bit整数（INT8）や4bit整数（INT4）で表現することで、モデルサイズを75〜87.5%削減し、推論速度を2〜4倍高速化できます。GPT-2（117M）ならfp32で~460MB → INT8で~115MBになります。

量子化の仕組み：「小数を整数に丸める」
  fp32 (32bit): -1.234567  →  4バイト
  fp16 (16bit): -1.234     →  2バイト  (2× 小さく)
  INT8  (8bit): -123       →  1バイト  (4× 小さく)  ← 量子化
  INT4  (4bit): -8〜7範囲  →  0.5バイト (8× 小さく) ← QLoRAで使用

  量子化式（対称量子化）:
    w_int = round(w_float / scale)
    scale = max(|W|) / 127    (INT8の場合、範囲 -127〜127)

  逆量子化（推論時の計算に使用）:
    w_dequant ≈ w_int × scale

  誤差は生じるが、LLMの重みは統計的に正規分布に近いため
  低ランクの変動は失われにくい → 品質への影響は限定的
Python — 量子化のスクラッチ実装（原理理解） 
import torch
import numpy as np

def quantize_int8(weight: torch.Tensor):
    """
    fp32 重みテンソルを INT8 に量子化する
    対称量子化（Symmetric Quantization）
    """
    # ① スケールを計算（最大絶対値 → INT8の範囲 127 にマッピング）
    scale = weight.abs().max() / 127.0

    # ② 整数に丸める（clamp で範囲外をクリップ）
    w_int8 = (weight / scale).round().clamp(-127, 127).to(torch.int8)

    return w_int8, scale

def dequantize_int8(w_int8: torch.Tensor, scale: float) -> torch.Tensor:
    """INT8 → fp32 に逆量子化"""
    return w_int8.to(torch.float32) * scale

# 動作確認
W = torch.randn(4, 4)
print("元の重み:\n", W)

print(f"\n量子化後（INT8）:\n{W_int8}")
print(f"スケール: {scale:.6f}")

print(f"\n逆量子化後:\n{W_dequant}")

error = (W - W_dequant).abs()
print(f"\n量子化誤差: 平均={error.mean():.6f}, 最大={error.max():.6f}")
# 量子化誤差: 平均=0.003..., 最大=0.007...（非常に小さい）

量子化の4つのアプローチ

① PTQ（Post-Training Quantization）: 学習済みモデルに後から量子化を適用。最も手軽。
② QAT（Quantization-Aware Training）: 学習中に量子化のシミュレーションを行い、量子化に強いモデルを学習。精度が最も高いが計算コスト大。
③ GPTQ: OBQアルゴリズムで重みを一つずつ量子化し、誤差を補正。LLMでの4bit量子化に有効。
④ AWQ: 活性化の大きさで重要なチャンネルを特定し、スケールを調整。GPTQより高速・同精度。現在の主流。

実用的な量子化の選び方
  • 開発・実験段階: load_in_8bit=True（transformers標準対応）→ 8bit、メモリ半減
  • 本番デプロイ（精度重視）: AWQ 4bit → サイズ75%削減、精度劣化最小
  • エッジ・モバイル: QAT + INT4/INT8 → Apple Silicon / Android NPU最適化
  • 量子化+ファインチューニング: QLoRA → 4bit量子化 + LoRA で最強の効率化
まとめ
  • 量子化: fp32→INT8で4倍、fp32→INT4で8倍のモデル圧縮。推論速度も2〜4倍向上
  • GPTQ/AWQは4bit量子化でfp16に近い精度を維持（perplexity劣化 <3%）
  • 実用上の定番: AWQ（速度・精度バランス）または QLoRA（ファインチューニング時）
  • 量子化誤差は正規分布的な重みには小さいため、LLMの大規模重みに特に有効

RAG — 検索拡張生成

LLMは学習データに含まれない最新情報や、ドメイン固有知識（社内文書・専門DB）を知ることができません。RAG（Retrieval-Augmented Generation）は「まず関連文書を検索し、それをコンテキストとしてLLMに渡す」ことで、ファインチューニングなしで知識を拡張できます。ChatGPT の検索機能やNotebookLMがこの仕組みを使っています。

   PDFs        512 token      text→768次元       FAISS / Chroma
   HTMLs       overlap=128    embedding model     (高速近傍探索)

  ─── オンライン（質問応答）────────────────────────────────────
  [質問] → [Embeddingに変換] → [ベクトルDB検索] → [上位k件取得]
   "○○とは"   query_vector     コサイン類似度↑     k=3〜5件

              → [プロンプト構築] → [LLM生成] → [回答]

                  質問:{query}"

Python — 高度なRAG：リランキング（Reranking）で精度向上 
## RAGの精度改善: 2段階検索（Retrieval → Reranking）
#
# 問題: Embeddingベースの検索は「意味的な近さ」を測るが、
#       細かいニュアンスを見落とすことがある
# 解決: Cross-Encoderで「クエリと文書の関連度」を精密に再評価（リランキング）

from sentence_transformers import CrossEncoder

# ─── Cross-Encoder: Q と文書を一緒にエンコードして精密に評価 ───
reranker = CrossEncoder("cross-encoder/ms-marco-MiniLM-L-6-v2")

def rag_with_reranking(query: str, first_k: int = 20, final_k: int = 3) -> list[str]:
    """
    2段階RAG:
    1. Embedding検索で候補を多め(k=20)に取得（Recall重視）
    2. Cross-Encoderで精密にリランキングして上位k=3を選ぶ（Precision重視）
    """
    # ① 第1段階: 粗い検索（高速だが精度は中程度）
    query_embed = embed_model.encode(["query: " + query], normalize_embeddings=True)
    _, indices = index.search(query_embed.astype(np.float32), min(first_k, len(documents)))
    candidates = [documents[i] for i in indices[0] if i < len(documents)]

    # ② 第2段階: Cross-Encoderでリランキング（遅いが精度高い）
    pairs = [[query, doc] for doc in candidates]
    scores = reranker.predict(pairs)

    # スコア順にソート
    ranked = sorted(zip(scores, candidates), reverse=True)
    top_docs = [doc for _, doc in ranked[:final_k]]

    print("リランキング後の上位文書:")
    for score, doc in ranked[:final_k]:
        print(f"  {score:.4f}: {doc[:50]}...")

    return top_docs

RAG vs ファインチューニング — どちらを使うべきか

RAGが有利な場面: 頻繁に更新される知識（ニュース・社内Wiki）、根拠の透明性が必要（出典を示せる）、大量の文書（数百万件）への対応、構築コストを抑えたい場合。
ファインチューニングが有利な場面: 特定のスタイル・トーンを学習させたい、応答速度が最優先、ドメイン特有の推論パターンを習得させたい場合。
RAG + ファインチューニングの組み合わせが最強（RAGで知識を供給し、FTでスタイルや推論能力を向上）。

まとめ
  • RAGの核心: テキスト→Embedding→ベクトルDB（FAISS/Chroma）→コサイン類似度検索→LLMへのコンテキスト注入
  • 精度向上の鍵はリランキング（Cross-Encoder）と適切なチャンク設計（サイズ・オーバーラップ）
  • ハルシネーション削減効果あり（「参考情報に基づいて回答」という制約を課せる）
  • 実用的なスタックは: LangChain/LlamaIndex（フレームワーク）+ Chroma/Pinecone（DB）+ e5/BGE（Embedding）

マルチモーダル — CLIP · Vision-Language Models

「この写真に猫が写っているか？」を判定するには、画像と「猫」というテキストを同じ意味空間に埋め込み、類似度を計算できれば良いです。CLIP（Contrastive Language-Image Pretraining）はOpenAIが2021年に発表したモデルで、4億枚の（画像, テキスト）ペアから対照学習によってこの共通埋め込み空間を学習しました。CLIPは現代のVision-Language Model（LLaVA, GPT-4V, Gemini等）の基盤技術です。

CLIPの対照学習：「正しいペアを近く、他は遠くに」
  ─── バッチ内の N=4 ペア ────────────────────────────────────

  画像エンコーダ(ViT)    テキストエンコーダ(Transformer)
  [犬の画像] → I₁       ["a photo of a dog"] → T₁
  [猫の画像] → I₂       ["a photo of a cat"] → T₂
  [車の画像] → I₃       ["a red sports car"] → T₃
  [海の画像] → I₄       ["ocean sunset view"] → T₄

  類似度行列 (N×N = 4×4):
             T₁    T₂    T₃    T₄
       I₁ [ 0.95  0.12  0.08  0.05 ]  ← (I₁,T₁) が正例
       I₂ [ 0.11  0.93  0.09  0.07 ]  ← (I₂,T₂) が正例
       I₃ [ 0.07  0.08  0.91  0.10 ]  ← (I₃,T₃) が正例
       I₄ [ 0.06  0.09  0.11  0.94 ]  ← (I₄,T₄) が正例

  損失: 対角成分（正例ペア）のスコアを最大化
        非対角成分（負例ペア）のスコアを最小化
        → InfoNCE Loss（対照損失）
$$\mathcal{L}_{\text{CLIP}} = -\frac{1}{N}\sum_{i=1}^{N} \log \frac{\exp(\text{sim}(I_i, T_i)/\tau)}{\sum_{j=1}^{N}\exp(\text{sim}(I_i, T_j)/\tau)}$$

$\tau$: 温度パラメータ（小さいほど分布がシャープ）/ sim: コサイン類似度 / N: バッチサイズ

Python — CLIPの対照損失スクラッチ実装 
import torch
import torch.nn as nn
import torch.nn.functional as F

class CLIPLoss(nn.Module):
    """
    CLIP の InfoNCE 対照損失
    画像-テキストペアを共通の埋め込み空間に引き寄せる
    """
    def __init__(self, temperature: float = 0.07):
        super().__init__()
        # 温度パラメータ: CLIPでは学習可能なパラメータとして実装
        self.log_temperature = nn.Parameter(torch.log(torch.tensor(temperature)))

    def forward(self, image_embeds: torch.Tensor, text_embeds: torch.Tensor) -> torch.Tensor:
        """
        Args:
            image_embeds: [N, D] — 正規化済み画像埋め込み
            text_embeds:  [N, D] — 正規化済みテキスト埋め込み
        Returns:
            対照損失のスカラー値
        """
        N = image_embeds.shape[0]

        # ① L2正規化（コサイン類似度 = 正規化済みベクトルの内積）
        image_embeds = F.normalize(image_embeds, dim=-1)
        text_embeds  = F.normalize(text_embeds,  dim=-1)

        # ② 類似度行列 [N, N]（温度でスケーリング）
        temperature = self.log_temperature.exp()
        logits = (image_embeds @ text_embeds.T) / temperature
        # logits[i,j] = sim(画像i, テキストj) / τ

        # ③ 正解ラベル: 対角成分が正例（i番目の画像とi番目のテキストがペア）
        labels = torch.arange(N, device=image_embeds.device)

        # ④ CrossEntropyLoss = Softmax + NLL Loss
        # 画像→テキスト方向と、テキスト→画像方向の両方を計算して平均
        loss_i2t = F.cross_entropy(logits,   labels)  # 各画像のペアテキストを当てる
        loss_t2i = F.cross_entropy(logits.T, labels)  # 各テキストのペア画像を当てる

        return (loss_i2t + loss_t2i) / 2

# 動作確認
N, D = 8, 512  # バッチサイズ8, 埋め込み次元512
image_embeds = torch.randn(N, D)
text_embeds  = torch.randn(N, D)

criterion = CLIPLoss(temperature=0.07)
loss = criterion(image_embeds, text_embeds)
print(f"CLIP Loss: {loss.item():.4f}")  # ランダムなので log(N) ≈ 2.08 付近

import torch
import torch.nn as nn

## LLaVA (Large Language and Vision Assistant) のアーキテクチャ
## = CLIP画像エンコーダ + 線形プロジェクション + LLM (Vicuna/LLaMA)
##
## 入力:  [画像] + [テキストの質問]
## 出力:  [テキストの回答]

class LLaVAProjector(nn.Module):
    """
    CLIP の画像埋め込みを LLM のテキスト空間に変換する
    「言語」と「視覚」の橋渡し役
    """
    def __init__(self, clip_dim: int = 768, llm_dim: int = 4096):
        super().__init__()
        # MLPプロジェクター（LLaVA 1.5以降はMLPが標準）
        self.projector = nn.Sequential(
            nn.Linear(clip_dim, llm_dim),
            nn.GELU(),
            nn.Linear(llm_dim, llm_dim)
        )

    def forward(self, image_features: torch.Tensor) -> torch.Tensor:
        """
        Args:
            image_features: [B, num_patches, clip_dim]
                           ViT-L/14 の場合: [B, 256, 1024]
        Returns:
            visual_tokens: [B, num_patches, llm_dim]
                          LLMのワード埋め込みと同じ次元
        """
        return self.projector(image_features)

Chain of Thought と推論モデル

「山田さんは100円持っていて50円のパンを買った。残りはいくら？」という問題を、LLMは「50円」と一発で答えるより、「100 - 50 = 50円です」と思考の過程を書いてから答える方が正確です。これがChain of Thought（CoT）プロンプティングです。OpenAIの o1/o3 モデルや DeepSeek-R1 は、この推論プロセスを長い内部独白（内部CoT）として実行することで、数学・論理・コーディングで人間トップレベルの性能を達成しました。

通常のPromptとChain of Thoughtの比較
  ─── 通常のプロンプト（誤りやすい）─────────────────────────
  Q: Juliaは5個のリンゴを持っている。Mikiに3個あげ、
     八百屋で7個買った。今何個持っている？
  A: 9個   ← 間違いやすい (5-3+7=9 は正しいが過程が不明瞭)

  ─── Few-Shot Chain of Thought ──────────────────────────────
  Q: Juliaは5個のリンゴを持っている。Mikiに3個あげ、
     八百屋で7個買った。今何個持っている？
  A: まずJuliaが最初に持っているリンゴは5個です。
     Mikiに3個あげると 5 - 3 = 2 個になります。
     その後7個買うと 2 + 7 = 9 個になります。
     答え: 9個   ← 正解（途中計算が検証可能）

  ─── Zero-Shot CoT（「ステップバイステップで考えて」魔法の一文）────
  Q: [問題] ステップバイステップで考えてください。
  A: [モデルが自動的に思考手順を展開する]
Python — Chain of Thought プロンプトの実装パターン 
from openai import OpenAI
client = OpenAI()  # OPENAI_API_KEY 環境変数が必要

def solve_with_cot(problem: str, mode: str = "zero_shot") -> str:
    """
    Chain of Thought プロンプティングで問題を解く

    Args:
        mode: "standard" / "zero_shot" / "few_shot"
    """
    if mode == "standard":
        # 通常: 直接回答を求める（推論プロセスなし）
        prompt = f"次の問題を解いてください:\n{problem}"

    elif mode == "zero_shot":
        # Zero-Shot CoT: 「ステップバイステップ」の一言を追加するだけ
        # Wei et al. (2022) が発見: この一文だけで性能が劇的に向上する!
        prompt = f"次の問題を解いてください:\n{problem}\n\nステップバイステップで考えてください。"

    elif mode == "few_shot":
        # Few-Shot CoT: 例題を見せてから解かせる（最も高精度）
        prompt = """以下の例を参考に、ステップバイステップで問題を解いてください。

例題:
Q: 農場に牛が15頭、鶏が8羽います。脚の合計は?
A: 牛は4本脚なので 15×4=60本。鶏は2本脚なので 8×2=16本。
   合計: 60+16=76本。答え: 76本

問題:
Q: """ + problem + "\nA:"

    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": prompt}],
        temperature=0.0  # 推論タスクは温度0で一貫性を確保
    )
    return response.choices[0].message.content

なぜ長い思考が性能を上げるのか：「計算量スケーリング則」

Kaplan et al. (2020) のスケーリング則では「モデルサイズ・データ量・計算量を増やすと性能が向上する」ことを示しました。o1/o3 が発見したのは テスト時計算量（Test-Time Compute）のスケーリングです。同じモデルでも、推論に使う計算量（トークン数）を増やすことで性能が向上し、より小さいモデルでも推論時間を増やせば大きいモデルと同等以上の性能が出せる場合があります。「考える時間」を与えれば良い、という直感的な発想の実装です。

CoT / 推論モデルを使うべき場面
  • 数学・論理パズル: 段階的な計算が必要。CoT / Self-Consistency が有効
  • コード生成・デバッグ: エラーの原因分析に推論モデル（o3/DeepSeek-R1）が有効
  • 科学的推論: 物理・化学の問題解決に思考連鎖が必要
  • 普通の会話・翻訳・要約: CoTは不要。Standard Promptで十分（コスト節約）
  • 速度優先タスク: 推論モデルは遅くコストが高い。リアルタイム応答には不向き
まとめ
  • CoT: 「ステップバイステップで考えてください」の一文で推論性能が大幅向上（Zero-Shot CoT）
  • Self-Consistency: temperature>0で複数回サンプリングして多数決 → さらに+5〜10%向上
  • Tree of Thoughts: 思考を木として探索し、行き詰まったら別経路を試みる（複雑な計画問題に有効）
  • o1/o3/DeepSeek-R1: 内部CoT（thinking tokens）をRLで強化学習 → テスト時計算量スケーリングで人間トップレベルの推論
  • 2025年のトレンド: 推論とツール使用（ReAct）の組み合わせ = AIエージェント
